# ⚡ Быстрое исправление HTTPS (если не помогло)

## Проблема: Все еще Mixed Content после установки APP_URL

Если вы установили `APP_URL=https://...` но ошибки остались, выполните следующие шаги:

### Шаг 1: Очистите ВСЕ кеши в Railway

В терминале Railway выполните **все команды подряд**:

```bash
php artisan config:clear
php artisan cache:clear
php artisan route:clear
php artisan view:clear
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

### Шаг 2: Перезапустите сервис

В Railway:
1. Найдите кнопку **"Restart"** или **"Redeploy"**
2. Нажмите на неё
3. Дождитесь полного перезапуска

### Шаг 3: Проверьте переменную APP_URL

В Railway:
1. Settings → Variables
2. Найдите `APP_URL`
3. Убедитесь, что значение **точно** такое:
   ```
   https://eventgo-production-8315.up.railway.app
   ```
   (без пробелов, с `https://` в начале)

4. Если значение неправильное - исправьте и сохраните
5. Перезапустите сервис снова

### Шаг 4: Очистите кеш браузера

1. Нажмите **Ctrl+Shift+Delete**
2. Выберите **"Все время"** (All time)
3. Отметьте **"Кешированные изображения и файлы"**
4. Нажмите **"Очистить данные"**
5. Закройте и откройте браузер заново
6. Откройте сайт в режиме инкогнито (Ctrl+Shift+N)

### Шаг 5: Проверьте в режиме инкогнито

Откройте сайт в режиме инкогнито (Ctrl+Shift+N) - это исключит влияние кеша браузера.

## 🔍 Диагностика

### Проверка 1: Правильно ли установлен APP_URL?

В терминале Railway выполните:

```bash
php artisan tinker
```

Затем в tinker:

```php
config('app.url')
```

Должно вернуть: `https://eventgo-production-8315.up.railway.app`

Если возвращает `http://...` - переменная не установлена правильно.

### Проверка 2: Проверьте, что HTTPS принудительно включен

В терминале Railway:

```bash
php artisan tinker
```

Затем:

```php
\URL::forceScheme('https');
asset('test.css')
```

Должно вернуть URL с `https://`.

## 🚨 Если ничего не помогло

### Вариант 1: Используйте secure_asset() напрямую

Временно замените все `asset()` на `secure_asset()` в views (но это не рекомендуется для постоянного решения).

### Вариант 2: Проверьте, что код обновлен

Убедитесь, что вы задеплоили последнюю версию кода с исправлениями:

```bash
git add .
git commit -m "Fix HTTPS"
git push
```

Railway автоматически пересоберет проект.

### Вариант 3: Проверьте логи

В Railway проверьте логи на наличие ошибок:

```bash
# В терминале Railway
tail -f /var/www/html/storage/logs/laravel.log
```

## ✅ После исправления

1. Откройте сайт
2. Нажмите F12 → Console
3. **Не должно быть** ошибок Mixed Content
4. Нажмите F12 → Network
5. Все файлы должны загружаться по **HTTPS** (зеленый замочек)

---

**Важно:** После каждого изменения переменных окружения в Railway:
1. Перезапустите сервис
2. Очистите кеш Laravel
3. Очистите кеш браузера
