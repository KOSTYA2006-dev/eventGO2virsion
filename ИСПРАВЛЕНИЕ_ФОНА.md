# 🔧 Исправление проблемы с задним фоном

## Проблема: Фон не работает

### Причины:
1. JavaScript файлы не загружаются (404 ошибка)
2. Скрипты выполняются до загрузки DOM
3. Элементы canvas/matrix-background не найдены
4. Ошибки в консоли браузера

## ✅ Решение

### Шаг 1: Проверьте консоль браузера

1. Откройте сайт
2. Нажмите **F12** (откроется консоль разработчика)
3. Перейдите на вкладку **Console**
4. Проверьте наличие ошибок:
   - Если видите `404` для файлов `backgraund.js` или `matrix.js` - файлы не загружаются
   - Если видите `Canvas element not found` - элемент не найден
   - Если видите другие ошибки - скопируйте их

### Шаг 2: Проверьте загрузку файлов

В консоли браузера (F12) перейдите на вкладку **Network**:
1. Обновите страницу (Ctrl+F5)
2. Найдите файлы:
   - `backgraund.js`
   - `matrix.js`
3. Проверьте статус:
   - ✅ **200** - файл загружен
   - ❌ **404** - файл не найден

### Шаг 3: Если файлы не загружаются (404)

#### В Railway:

1. **Проверьте, что файлы в репозитории:**
   ```bash
   # Убедитесь, что папка public/assets/js/ существует
   ls public/assets/js/
   ```

2. **Пересоберите Docker образ:**
   - Сделайте новый commit и push в GitHub
   - Railway автоматически пересоберет образ

3. **Проверьте права доступа:**
   ```bash
   # В терминале Railway
   chmod -R 755 /var/www/html/public/assets
   chown -R www-data:www-data /var/www/html/public/assets
   ```

#### В Docker локально:

```bash
docker-compose exec app chmod -R 755 /var/www/html/public/assets
docker-compose exec app chown -R www-data:www-data /var/www/html/public/assets
docker-compose restart app
```

### Шаг 4: Очистите кеш браузера

1. Нажмите **Ctrl+Shift+Delete**
2. Выберите "Кешированные изображения и файлы"
3. Очистите кеш
4. Обновите страницу (**Ctrl+F5**)

### Шаг 5: Проверьте элементы на странице

В консоли браузера (F12) выполните:

```javascript
// Проверка canvas
console.log(document.getElementById('c'));

// Проверка matrix-background
console.log(document.getElementById('matrix-background'));
```

Если оба элемента найдены - проблема в скриптах.
Если `null` - элементы не созданы в HTML.

## 🔍 Диагностика

### Проверка 1: Файлы доступны?

Откройте в браузере:
- `https://ваш-домен.railway.app/assets/js/backgraund.js`
- `https://ваш-домен.railway.app/assets/js/matrix.js`

Если файлы открываются - они доступны.
Если 404 - файлы не загружены в Docker образ.

### Проверка 2: Элементы на странице?

В консоли браузера:
```javascript
document.getElementById('c') // должен вернуть <canvas>
document.getElementById('matrix-background') // должен вернуть <div>
```

### Проверка 3: Скрипты выполняются?

В консоли браузера проверьте наличие ошибок JavaScript.

## 🚀 Быстрое исправление

Если ничего не помогло:

1. **Пересоберите проект:**
   ```bash
   # Сделайте commit и push
   git add .
   git commit -m "Fix background scripts"
   git push
   ```

2. **В Railway перезапустите сервис:**
   - Settings → Restart

3. **Очистите кеш:**
   ```bash
   # В терминале Railway
   php artisan config:clear
   php artisan cache:clear
   php artisan view:clear
   ```
