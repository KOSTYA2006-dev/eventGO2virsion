# Полная инструкция по настройке YooKassa для EventGo

## ✅ Что уже готово в проекте:

1. ✅ Интеграция YooKassa полностью реализована
2. ✅ Автоматическая обработка оплаты через webhook
3. ✅ Автоматическая отправка чека и билета на email после оплаты
4. ✅ Админка для просмотра всех покупок и информации о покупателях
5. ✅ Отображение статусов отправки чека и билета в админке

## 📋 Шаг 1: Регистрация в YooKassa

1. Перейдите на https://yookassa.ru/
2. Зарегистрируйтесь или войдите в личный кабинет
3. Заполните данные о вашей организации/ИП
4. Дождитесь модерации (обычно 1-3 дня)

## 🔑 Шаг 2: Получение данных для подключения

1. Войдите в личный кабинет YooKassa: https://yookassa.ru/my
2. Перейдите в раздел **"Настройки"** → **"API"**
3. Скопируйте следующие данные:
   - **Shop ID** (идентификатор магазина) - например: `123456`
   - **Секретный ключ** (Secret Key) - длинная строка символов

## ⚙️ Шаг 3: Настройка переменных окружения

Откройте файл `.env` в корне проекта и добавьте/обновите следующие строки:

```env
# Настройки YooKassa
YOOKASSA_SHOP_ID=ваш_shop_id_здесь
YOOKASSA_SECRET_KEY=ваш_секретный_ключ_здесь
YOOKASSA_TEST_MODE=true
USE_YOOKASSA=true
```

**Важно:**
- `YOOKASSA_TEST_MODE=true` - для тестирования (не списывает реальные деньги)
- `YOOKASSA_TEST_MODE=false` - для продакшена (реальные платежи)
- `USE_YOOKASSA=true` - включает использование YooKassa вместо ручной оплаты

## 🔔 Шаг 4: Настройка Webhook

Webhook нужен для автоматической обработки оплаты и отправки билетов.

### Для продакшена (реальный сайт):

1. Войдите в личный кабинет YooKassa: https://yookassa.ru/my
2. Перейдите в раздел **"Настройки"** → **"Уведомления"**
3. Нажмите **"Добавить URL"**
4. Введите URL вашего webhook:
   ```
   https://ваш-домен.ru/payment/webhook
   ```
   Например: `https://eventgo.ru/payment/webhook`
5. Выберите события для уведомлений:
   - ✅ `payment.succeeded` - успешная оплата (обязательно!)
   - ✅ `payment.canceled` - отмена оплаты (опционально)
6. Сохраните настройки

### Для локальной разработки:

Используйте ngrok для создания туннеля:

1. Установите ngrok: https://ngrok.com/
2. Запустите локальный сервер: `php artisan serve`
3. В другом терминале запустите ngrok:
   ```bash
   ngrok http 8000
   ```
4. Скопируйте HTTPS URL (например: `https://abc123.ngrok.io`)
5. В YooKassa добавьте webhook URL:
   ```
   https://abc123.ngrok.io/payment/webhook
   ```

## 📧 Шаг 5: Настройка отправки email

Для отправки билетов и чеков на email покупателей настройте почту.

### Для Gmail:

```env
MAIL_MAILER=smtp
MAIL_HOST=smtp.gmail.com
MAIL_PORT=587
MAIL_USERNAME=ваш_email@gmail.com
MAIL_PASSWORD=ваш_пароль_приложения
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=ваш_email@gmail.com
MAIL_FROM_NAME="EventGo"
```

**Как получить пароль приложения Gmail:**
1. Перейдите: https://myaccount.google.com/apppasswords
2. Войдите в аккаунт Google
3. Выберите "Почта" и "Другое устройство"
4. Введите название: "EventGo"
5. Скопируйте сгенерированный пароль (16 символов)
6. Используйте этот пароль в `MAIL_PASSWORD`

### Для Yandex:

```env
MAIL_MAILER=smtp
MAIL_HOST=smtp.yandex.ru
MAIL_PORT=465
MAIL_USERNAME=ваш_email@yandex.ru
MAIL_PASSWORD=ваш_пароль
MAIL_ENCRYPTION=ssl
MAIL_FROM_ADDRESS=ваш_email@yandex.ru
MAIL_FROM_NAME="EventGo"
```

### Для Mail.ru:

```env
MAIL_MAILER=smtp
MAIL_HOST=smtp.mail.ru
MAIL_PORT=465
MAIL_USERNAME=ваш_email@mail.ru
MAIL_PASSWORD=ваш_пароль
MAIL_ENCRYPTION=ssl
MAIL_FROM_ADDRESS=ваш_email@mail.ru
MAIL_FROM_NAME="EventGo"
```

## 🧪 Шаг 6: Тестирование

### Тестовые карты YooKassa:

**Успешная оплата:**
- Номер карты: `5555555555554444`
- Срок действия: любая будущая дата (например: 12/25)
- CVV: любые 3 цифры (например: 123)
- Имя держателя: любое

**Отклоненная оплата:**
- Номер карты: `5555555555554477`
- Остальное аналогично

### Процесс тестирования:

1. **Создайте тестовый заказ:**
   - Перейдите на главную страницу сайта
   - Выберите билет
   - Заполните форму покупателя
   - Нажмите "Оформить заказ"

2. **Оплата:**
   - Вы будете перенаправлены на страницу YooKassa
   - Введите данные тестовой карты
   - Подтвердите оплату

3. **Проверка:**
   - После оплаты вы вернетесь на страницу успеха
   - Проверьте email покупателя - должны прийти:
     - ✅ Чек об оплате
     - ✅ Билет на мероприятие
   - В админке (`/admin/orders`) проверьте:
     - ✅ Статус заказа: "Оплачен"
     - ✅ Чек отправлен: ✓ Да
     - ✅ Билет отправлен: ✓ Да

## 📊 Шаг 7: Проверка админки

1. Войдите в админку: `/admin/login`
   - Email: `admin@eventgo.ru`
   - Пароль: `admin123`

2. Перейдите в раздел **"Заказы"**

3. Вы увидите:
   - Все заказы с информацией о покупателях
   - Статусы оплаты
   - Статусы отправки чека и билета
   - Email покупателей
   - ID платежа YooKassa (для оплаченных заказов)

4. Нажмите на заказ для просмотра деталей:
   - Полная информация о покупателе (ФИО, телефон, email, вид деятельности)
   - Информация о билете
   - Информация об оплате
   - Статусы отправки чека и билета
   - Ссылки для скачивания чека и билета (если они были сгенерированы)

## 🔍 Проверка работы системы

### Логи:

Все операции логируются в файл `storage/logs/laravel.log`:

```bash
# Просмотр логов в реальном времени
tail -f storage/logs/laravel.log
```

Что логируется:
- Создание платежей YooKassa
- Получение webhook от YooKassa
- Отправка email (чек и билет)
- Ошибки

### Проверка webhook:

1. В админке YooKassa перейдите в **"Настройки"** → **"Уведомления"**
2. Проверьте, что webhook URL указан правильно
3. Нажмите **"Отправить тестовое уведомление"**
4. Проверьте логи на сервере - должно появиться сообщение о получении webhook

## ⚠️ Важные замечания

1. **Тестовый режим:**
   - В тестовом режиме (`YOOKASSA_TEST_MODE=true`) реальные деньги не списываются
   - Для продакшена установите `YOOKASSA_TEST_MODE=false`

2. **Webhook:**
   - Webhook URL должен быть доступен из интернета
   - Используйте HTTPS для продакшена
   - Проверьте, что webhook не блокируется файрволом

3. **Email:**
   - Проверьте настройки email перед запуском в продакшен
   - Убедитесь, что письма не попадают в спам
   - Проверьте лимиты отправки на вашем почтовом сервисе

4. **Безопасность:**
   - Никогда не публикуйте `YOOKASSA_SECRET_KEY` в публичных репозиториях
   - Используйте переменные окружения для всех секретных данных
   - Регулярно проверяйте логи на наличие ошибок

## 🚀 Переход в продакшен

Когда все протестировано и работает:

1. **В YooKassa:**
   - Завершите модерацию аккаунта
   - Переключитесь на боевой режим

2. **В .env:**
   ```env
   YOOKASSA_TEST_MODE=false
   APP_ENV=production
   APP_DEBUG=false
   ```

3. **Очистите кеш:**
   ```bash
   php artisan config:clear
   php artisan cache:clear
   php artisan config:cache
   ```

4. **Проверьте webhook:**
   - Убедитесь, что webhook URL указывает на продакшен домен
   - Протестируйте тестовым уведомлением
