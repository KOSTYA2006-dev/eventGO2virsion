# ⚡ Быстрое исправление проблем после деплоя

## Проблемы: Пропали билеты, фон и верстка

### 🚀 Решение через веб-интерфейс Railway (БЕЗ CLI):

1. **Откройте ваш проект в Railway**
2. **Выберите ваш Web Service**
3. **Перейдите в раздел "Deployments"**
4. **Нажмите на последний деплой** (самый верхний)
5. **Нажмите кнопку "View Logs"** или найдите кнопку "Shell" / "Terminal"
6. **Выполните команды по одной:**

```bash
php artisan migrate --force
php artisan db:seed --force
php artisan config:clear
php artisan cache:clear
php artisan view:clear
php artisan config:cache
php artisan route:cache
```

**Или выполните все сразу через скрипт:**

```bash
# Скопируйте содержимое файла fix-deployment.sh и выполните
bash -c "php artisan migrate --force && php artisan db:seed --force && php artisan config:clear && php artisan cache:clear && php artisan view:clear && php artisan config:cache && php artisan route:cache"
```

### 🔧 Альтернатива: Через SSH/Shell в Railway

1. В Railway откройте ваш Web Service
2. Найдите кнопку **"Shell"** или **"Connect"** (обычно в правом верхнем углу)
3. Откроется терминал - выполните команды выше

### 📝 Или установите Railway CLI (опционально):

```bash
# Windows (PowerShell)
iwr https://railway.app/install.ps1 -useb | iex

# Mac/Linux
curl -fsSL https://railway.app/install.sh | sh

# Затем
railway login
railway link
railway run php artisan migrate --force
railway run php artisan db:seed --force
```

3. **Перезапустите сервис** в Railway (Settings → Restart)

### 🐳 Быстрое решение (Docker):

```bash
# Выполните миграции и создайте билеты
docker-compose exec app php artisan migrate --force
docker-compose exec app php artisan db:seed --force

# Очистите кеш
docker-compose exec app php artisan config:clear
docker-compose exec app php artisan cache:clear
docker-compose exec app php artisan view:clear

# Исправьте права доступа
docker-compose exec app chmod -R 755 /var/www/html/public/assets
docker-compose exec app chown -R www-data:www-data /var/www/html/public/assets

# Перезапустите
docker-compose restart app
```

### ✅ Что должно произойти:

- ✅ Появятся билеты на главной странице
- ✅ Заработает Matrix фон
- ✅ Верстка вернется в норму

### 🔍 Если не помогло:

1. Проверьте логи: `railway logs` или `docker-compose logs app`
2. Убедитесь, что файлы `public/assets/` существуют в репозитории
3. Проверьте переменные окружения в Railway
4. Пересоберите Docker образ (новый push в GitHub для Railway)

---

**После обновления `railway.json`** - при следующем деплое миграции и seeders будут выполняться автоматически!
