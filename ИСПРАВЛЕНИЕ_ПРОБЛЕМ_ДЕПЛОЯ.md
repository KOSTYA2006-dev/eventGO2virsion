# 🔧 Исправление проблем после деплоя

## Проблема: Пропали билеты, фон и верстка

### Решение 1: Выполните миграции и seeders

После деплоя нужно выполнить миграции и создать начальные данные (билеты, админ и т.д.).

#### В Railway:

1. Откройте ваш Web Service в Railway
2. Перейдите в раздел "Settings" → "Deploy"
3. Найдите раздел "Run Command" или используйте Railway CLI:

```bash
# Установите Railway CLI (если еще не установлен)
npm i -g @railway/cli

# Войдите в Railway
railway login

# Подключитесь к проекту
railway link

# Выполните миграции и seeders
railway run php artisan migrate --force
railway run php artisan db:seed --force
```

Или через веб-интерфейс Railway:
- Откройте ваш Web Service
- Перейдите в "Deployments"
- Нажмите на последний деплой
- Откройте "View Logs" и выполните команды через терминал

#### Через Docker:

```bash
docker-compose exec app php artisan migrate --force
docker-compose exec app php artisan db:seed --force
```

### Решение 2: Проверьте статические файлы

Убедитесь, что статические файлы (CSS, JS, изображения, шрифты) доступны.

#### Проверьте в браузере:

Откройте консоль разработчика (F12) и проверьте:
- Загружаются ли файлы из `/assets/css/main-set.css`
- Загружаются ли файлы из `/assets/js/backgraund.js`
- Загружаются ли шрифты из `/assets/text-style/Ubuntu/`

Если файлы не загружаются (404 ошибка):

1. **Проверьте, что файлы скопированы в Docker образ:**
   - В Railway: проверьте логи сборки
   - Убедитесь, что папка `public/assets/` существует в репозитории

2. **Проверьте права доступа:**
   ```bash
   # В Docker
   docker-compose exec app chmod -R 755 /var/www/html/public/assets
   docker-compose exec app chown -R www-data:www-data /var/www/html/public/assets
   ```

3. **Очистите кеш:**
   ```bash
   # В Railway
   railway run php artisan config:clear
   railway run php artisan cache:clear
   railway run php artisan view:clear
   
   # В Docker
   docker-compose exec app php artisan config:clear
   docker-compose exec app php artisan cache:clear
   docker-compose exec app php artisan view:clear
   ```

### Решение 3: Проверьте конфигурацию Nginx

Убедитесь, что Nginx правильно обслуживает статические файлы. Конфигурация уже обновлена в `docker/nginx.conf`.

Если проблемы остаются, пересоберите Docker образ:

```bash
# В Railway - просто сделайте новый деплой (push в GitHub)
# В Docker локально:
docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

### Решение 4: Проверьте переменные окружения

Убедитесь, что все необходимые переменные окружения установлены:

```env
APP_ENV=production
APP_DEBUG=false
APP_URL=https://ваш-домен.railway.app
DB_CONNECTION=mysql
DB_HOST=${{RAILWAY_PRIVATE_DOMAIN}}
DB_PORT=3306
DB_DATABASE=${{MYSQL_DATABASE}}
DB_USERNAME=root
DB_PASSWORD=${{MYSQL_ROOT_PASSWORD}}
```

### Решение 5: Проверьте логи

Проверьте логи приложения на наличие ошибок:

```bash
# В Railway
railway logs

# В Docker
docker-compose logs -f app
```

### Быстрое исправление (все сразу):

```bash
# 1. Очистка кеша
php artisan config:clear
php artisan cache:clear
php artisan route:clear
php artisan view:clear

# 2. Миграции и seeders
php artisan migrate --force
php artisan db:seed --force

# 3. Оптимизация для production
php artisan config:cache
php artisan route:cache
php artisan view:cache

# 4. Проверка прав доступа (в Docker)
chmod -R 755 storage bootstrap/cache public/assets
chown -R www-data:www-data storage bootstrap/cache public/assets
```

## После исправления

1. **Проверьте главную страницу** - должны появиться билеты
2. **Проверьте фон** - должен работать Matrix эффект
3. **Проверьте верстку** - стили должны загружаться правильно

## Если проблемы остаются

1. Проверьте, что все файлы закоммичены в Git и задеплоены
2. Убедитесь, что база данных создана и доступна
3. Проверьте, что миграции выполнились успешно (проверьте таблицы в БД)
4. Проверьте логи на наличие ошибок PHP

---

**Важно:** После каждого деплоя Railway автоматически выполнит миграции и seeders благодаря обновленному `railway.json`. Но если вы уже задеплоили до обновления, выполните команды вручную.
