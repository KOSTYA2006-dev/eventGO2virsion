# 🚀 Быстрый деплой EventGo

Этот документ содержит инструкции по быстрому деплою приложения на различные платформы.

## 📦 Вариант 1: Docker (Локально или на VPS)

### Быстрый старт с Docker Compose

1. **Создайте файл `.env`** (скопируйте из `.env.example` и настройте):
```bash
cp .env.example .env
```

2. **Настройте переменные окружения в `.env`**:
```env
APP_ENV=production
APP_DEBUG=false
APP_URL=http://your-domain.com

DB_CONNECTION=mysql
DB_HOST=db
DB_DATABASE=eventgo
DB_USERNAME=eventgo
DB_PASSWORD=your_secure_password

# Остальные настройки (почта, платежи и т.д.)
```

3. **Запустите приложение**:
```bash
docker-compose up -d
```

4. **Выполните миграции**:
```bash
docker-compose exec app php artisan migrate --force
docker-compose exec app php artisan db:seed
```

5. **Приложение будет доступно на** `http://localhost:8000`

### Деплой на VPS с Docker

1. Скопируйте проект на сервер
2. Настройте `.env`
3. Запустите `docker-compose up -d`
4. Настройте домен через Nginx reverse proxy (опционально)

## ☁️ Вариант 2: Railway (Рекомендуется - самый быстрый)

**Railway** - один из самых простых способов задеплоить Laravel приложение.

### Шаги:

1. **Зарегистрируйтесь на** [railway.app](https://railway.app)

2. **Создайте новый проект** → "Deploy from GitHub repo"

3. **Добавьте сервисы**:
   - **MySQL** - база данных
   - **Web Service** - ваше приложение

4. **Настройте переменные окружения** в Railway:
   ```
   APP_ENV=production
   APP_DEBUG=false
   APP_KEY=your-generated-key
   DB_CONNECTION=mysql
   DB_HOST=${{RAILWAY_PRIVATE_DOMAIN}}
   DB_PORT=3306
   DB_DATABASE=${{MYSQL_DATABASE}}
   DB_USERNAME=root
   DB_PASSWORD=${{MYSQL_ROOT_PASSWORD}}
   ```
   
   **Примечание:** Railway автоматически создаст переменные `MYSQL_DATABASE`, `MYSQL_ROOT_PASSWORD` и `RAILWAY_PRIVATE_DOMAIN` при добавлении MySQL сервиса. Вы можете использовать их напрямую или через `${{MySQL.MYSQLHOST}}` формат.

5. **Создайте `railway.json`** в корне проекта:
```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "DOCKERFILE",
    "dockerfilePath": "Dockerfile"
  },
  "deploy": {
    "startCommand": "php artisan migrate --force && /usr/local/bin/start.sh",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

6. **Railway автоматически задеплоит** при каждом push в GitHub!

**Преимущества:**
- ✅ Бесплатный тариф (с ограничениями)
- ✅ Автоматический деплой из GitHub
- ✅ Встроенная база данных
- ✅ HTTPS из коробки
- ✅ Очень простой интерфейс

## 🌐 Вариант 3: Render

**Render** - еще один отличный вариант для быстрого деплоя.

### Шаги:

1. **Зарегистрируйтесь на** [render.com](https://render.com)

2. **Создайте новый Web Service** → "Build and deploy from a Git repository"

3. **Настройки сборки**:
   - **Build Command**: `composer install --no-dev --optimize-autoloader && npm ci && npm run build`
   - **Start Command**: `php artisan migrate --force && php artisan serve --host=0.0.0.0 --port=$PORT`

4. **Добавьте PostgreSQL Database** (отдельный сервис)

5. **Настройте переменные окружения**:
   ```
   APP_ENV=production
   APP_DEBUG=false
   DB_CONNECTION=pgsql
   DB_HOST=<your-db-host>
   DB_DATABASE=<your-db-name>
   DB_USERNAME=<your-db-user>
   DB_PASSWORD=<your-db-password>
   ```

**Преимущества:**
- ✅ Бесплатный тариф
- ✅ Автоматический деплой
- ✅ HTTPS из коробки

## 🚁 Вариант 4: Fly.io

**Fly.io** - отличный вариант для деплоя с Docker.

### Шаги:

1. **Установите flyctl**:
```bash
# Windows
iwr https://fly.io/install.ps1 -useb | iex
```

2. **Войдите в Fly.io**:
```bash
fly auth login
```

3. **Создайте приложение**:
```bash
fly launch
```

4. **Настройте `fly.toml`** (создастся автоматически):
```toml
app = "your-app-name"
primary_region = "ams"  # или другой регион

[build]

[env]
  APP_ENV = "production"
  APP_DEBUG = "false"

[[services]]
  internal_port = 80
  protocol = "tcp"
```

5. **Добавьте PostgreSQL**:
```bash
fly postgres create
fly postgres attach <postgres-app-name> -a <your-app-name>
```

6. **Деплой**:
```bash
fly deploy
```

## 🔧 Вариант 6: VPS с Docker (Vultr, DigitalOcean, Hetzner)

Если у вас есть VPS, самый быстрый способ - использовать Docker:

1. **Установите Docker и Docker Compose на сервер**:

3. **Настройте `.env`**

4. **Запустите**:
```bash
docker-compose up -d
docker-compose exec app php artisan migrate --force
```

5. **Настройте домен** (опционально, через Nginx reverse proxy)

## 📝 Общие шаги для всех платформ

После деплоя обязательно:

1. **Сгенерируйте APP_KEY** (если не установлен):
```bash
php artisan key:generate
```

2. **Выполните миграции**:
```bash
php artisan migrate --force
```

3. **Создайте администратора** (если нужно):
```bash
php artisan db:seed
```

4. **Настройте переменные окружения**:
   - `APP_URL` - URL вашего сайта
   - `MAIL_*` - настройки почты
   - `VTB_*` - реквизиты для платежей
   - `EVENT_DATE` - дата мероприятия
